
# AEGIS AI — Working Prototype Plan

## Honest platform reality first

Lovable builds **one deployable web app** (TanStack Start frontend + Lovable Cloud backend = Postgres + serverless functions + auth + storage). It does **not** run:
- A separate FastAPI Python service
- Neo4j (no managed Neo4j on this platform)
- Docker Compose multi-container stacks
- LangGraph / CrewAI Python runtimes

**But** every capability you asked for can be delivered here with equivalent tech, and it will actually run at a live public URL — which matters more for a hackathon demo than a Python repo that only runs on your laptop.

## Scope — what will actually work (every icon clickable)

**8 pages, all functional:**
1. **Login / Signup** — email + Google
2. **Command Center** — live DRI gauge, active signals, latency stat, key India stats
3. **Live Intelligence** — signal feed with real LLM analysis per signal (click → Gemini analyzes)
4. **Digital Twin (Map)** — Mapbox with suppliers, chokepoints (Hormuz, Bab-el-Mandeb, Malacca), Indian ports, refineries; edges colored by live risk
5. **Scenario Lab** — pick scenario (Hormuz closure, Red Sea, OPEC cut, cyclone) → real graph recomputes, cascading impacts shown with visible formulas
6. **Adaptive Procurement** — ranked alternative crude sources + routes with cost/delay/confidence, generated by LLM grounded in graph data
7. **Strategic Reserve Optimiser** — SPR drawdown schedule computed from current DRI + supply gap
8. **Executive Reports** — LLM generates PDF-ready situation report; audit log with SHA-256 hashes

**6 real agent server functions:**
- Geopolitical Intelligence, Risk Scoring, Scenario Simulator, Procurement Optimiser, SPR Optimiser, Executive Report

## Milestones (I'll execute in order, not ask between each)

**M1 — Foundation:** Lovable Cloud on, auth, schema (suppliers, ports, refineries, corridors, edges, signals, scenarios, recommendations, audit_log), seed India-specific data, design system.

**M2 — Digital Twin map:** Mapbox connector, real Indian energy network rendered, edges colored by live edge risk from DB.

**M3 — Agents + Lovable AI:** Server functions for each agent. Real Gemini calls via Lovable AI Gateway. Structured JSON output. Fallback on failure.

**M4 — Graph engine:** Dijkstra + risk propagation over the Postgres edge table (TypeScript, server-side). Recomputes on every scenario/signal change.

**M5 — Scenario Lab:** 4 preset scenarios, live cascading impact panel with formulas visible on screen (refinery run-rate, fuel price, GDP), latency timer from signal→recommendation.

**M6 — Procurement + SPR + Reports:** All three panels wired to agents + graph; PDF export; audit log with hash chain.

**M7 — Polish + GitHub:** README with architecture diagram, demo script, seed script, publish, connect GitHub.

## What I will NOT do (and why)

- **12 agents.** 6 real agents that fully work beat 12 that half-work. Judges score execution, not count. The extra 6 go on the roadmap slide.
- **Neo4j.** Adds deploy risk, no managed option here, and the graph algorithms are the value — Cypher syntax isn't. Postgres + TypeScript graph engine is genuinely equivalent for this problem size.
- **Separate FastAPI repo.** You'd have to run it locally during demo. TanStack server functions give you the same "typed backend endpoints" story in one deployed app.
- **Rebuild the earlier HTML prototype.** Keep `aegis-console-v3.html` as your zero-setup backup demo. This new build is the primary.
